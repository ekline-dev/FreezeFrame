# Security Policy

## Supported Versions

Since FreezeFrame is currently in active development, only the latest release is supported with security updates. If you are using a beta or an older build, please update to the latest version before reporting an issue.

| Version | Supported | Notes |
| 0.2.0 | :construction: | In Development (New Features) |
| 0.1.0 | :white_check_mark: | Current Stable Release |
| 0.0.0 | :x: | Prototype (Lots of Bugs) |


## Reporting a Vulnerability

Security is a priority, and we appreciate the community's help in keeping FreezeFrame safe. If you discover a security vulnerability, please do **not** open a public issue on the repository. 

Instead, please report it privately:

1. Send an email to **[CoreByteInteractions@outlook.com]** with the subject line `[SECURITY] FreezeFrame Vulnerability`.
2. Please include a detailed description of the issue, the steps required to reproduce it, and any relevant logs or screenshots.
3. You can expect an initial acknowledgment of your report within 48 hours. 

If the vulnerability is verified, a patch will be developed and released as quickly as possible. We will keep you updated on the progress and, with your permission, credit you in the release notes once the patch is live.
